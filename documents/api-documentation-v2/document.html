API Documentation v2.0

The official guide to integrating with our services.

Welcome to the v2.0 API. This version introduces new endpoints, improved authentication, and a more consistent data structure. All v1 endpoints will be deprecated on December 31, 2024.

Authentication

All requests to the API must be authenticated using an API key. Include your key in the Authorization header as a Bearer token.

Authorization: Bearer YOUR_API_KEY

Endpoints

GET /users/{id}

Retrieves a user object by its unique ID.

Parameters:

  • id (string, required): The ID of the user to retrieve.

Example Response:

{
  "id": "usr_12345",
  "name": "Jane Doe",
  "email": "jane.doe@example.com",
  "created_at": "2024-05-20T10:00:00Z"
}

POST /widgets

Creates a new widget.

Request Body:

{
  "name": "My New Widget",
  "color": "blue",
  "inventory": 100
}
